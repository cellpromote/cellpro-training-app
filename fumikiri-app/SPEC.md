## アプリの概要

**踏切シミュレーター** は、踏切の動作をブラウザ上で再現するインタラクティブなアプリです。

- 遮断機の開閉
- 警告灯の点滅
- 電車の走行アニメーション
- 踏切警報音・電車の走行音

の4要素が連動して動作します。

---

## 使用技術

| 技術 | バージョン | 用途 |
|------|-----------|------|
| **HTML5** | — | アプリ全体の骨格 |
| **CSS3** | — | ボタン・タイトルのスタイリング |
| **JavaScript (ES6+)** | — | ゲームロジック全般 |
| **[Phaser 3](https://phaser.io/)** | 3.60.0 | 2Dゲームフレームワーク（描画・アニメーション管理） |
| **Web Audio API** | ブラウザ標準 | 効果音の合成・再生 |

### Phaser 3 とは？

Phaser は、ブラウザ上で動作する **2D ゲームフレームワーク** です。  
このアプリでは「ゲーム」というよりも **Canvas への描画エンジン** として活用しています。

主に使っている機能：
- `Phaser.Scene` — 毎フレーム呼ばれる `create()` と `update()` の管理
- `Graphics` オブジェクト — 図形（四角・円・線）をコードで描画
- `ScaleManager` — 画面サイズに合わせたCanvas自動スケーリング

### Web Audio API とは？

ブラウザ標準の音声合成 API です。  
音声ファイルを用意しなくても、**プログラムで音を生成**できます。  
このアプリでは、踏切ベルと電車の走行音をすべてコードで生成しています。

```
fumikiri-app/
└── index.html   ← HTML・CSS・JavaScript がすべて1ファイルに収まっています
```

**シングルファイル構成**を採用しています。

## 機能仕様

### 1. 遮断機の操作（「遮断機を操作」ボタン）

| 状態 | 動作 |
|------|------|
| ゲート開 → 閉 | アームが水平（0°）から斜め（-85°）へ回転 / 警告灯・ベル音が開始 |
| ゲート閉 → 開 | アームが斜め（-85°）から水平（0°）へ回転 / 警告灯・ベル音が停止 |

アームの回転は毎フレーム **7% ずつ目標角度に近づく** イージングアニメーションです。

```js
// イージングの例（update() 内で毎フレーム実行）
armAngle += (armAngleTarget - armAngle) * 0.07;
```

### 2. 警告灯の点滅

- 左右のボックスに合計 **4 つの LED** があります
- 350ms ごとに **左2つ → 右2つ** が交互に点滅します
- 各ライトには `phase`（位相）プロパティがあり、点滅タイミングをずらしています

### 3. 電車の走行（「電車を走らせる」ボタン）

電車の走行は以下の順序で進行します：

```
ボタン押下
  ↓
遮断機が自動的に閉まる
  ↓
電車が画面左から登場（速度0から加速）
  ↓
最高速度 4.5px/フレーム で通過
  ↓
画面右端を出ると速度0にリセット
  ↓
遮断機が自動的に開く
```

電車は 4 両編成で、車体・窓・車輪・ヘッドライトが `Graphics` で描かれています。

### 4. ボタン操作中の制御

| 条件 | 制御 |
|------|------|
| 電車走行中 | 「遮断機を操作」ボタンは無効（`disabled`） |
| 電車走行中 | 「電車を走らせる」ボタンは無効（`disabled`） |

## 状態管理

グローバル変数でアプリの状態を管理しています。

```js
// 主要な状態変数
let armAngle = 0;         // 現在のアーム角度（描画に使用）
let armAngleTarget = 0;   // 目標のアーム角度（0 or -85）
let gateOpen = true;      // ゲートが開いているか
let warningActive = false;// 警告中か
let trainX = -300;        // 電車のX座標
let trainSpeed = 0;       // 電車の速度
let trainRunning = false; // 電車が走行中か
```

Phaser の `update()` が毎フレーム呼ばれ、これらの状態をもとに描画が更新されます。

```
状態変数 → update() で計算 → Graphics.clear() + 再描画
```

## 音声の仕組み

音声ファイルは一切なく、すべて **Web Audio API** でプログラム生成しています。

### 踏切ベル音

```
3つのサイン波オシレーター（1480Hz / 2960Hz / 4440Hz）
  ↓
700ms ごとに短く鳴らす
  ↓
「カンカンカン」に聞こえる
```

### 電車の走行音

```
ホワイトノイズを生成
  ↓
ローパスフィルター（180Hz 以下のみ通過）= 低いゴーッという音
  ↓
1.5秒でフェードイン → 電車が去ると1秒でフェードアウト
```

**`getAudioCtx()` 関数** は AudioContext を遅延生成します。  
ブラウザのポリシー上、ユーザー操作（ボタンクリック）後でないと音声を再生できないためです。

## レスポンシブ対応

| 技術 | 内容 |
|------|------|
| Phaser `ScaleManager` の `FIT` モード | Canvas を論理サイズ 700×360 のまま画面に合わせてスケーリング |
| CSS `clamp()` | ボタン・タイトルのフォントサイズを画面幅に合わせて可変 |
| `safe-area-inset` | ノッチ付きスマートフォン（iPhone等）への対応 |
| タッチターゲット 44px 以上 | スマートフォンでも押しやすいボタンサイズ |

## コードを読む際のヒント

### どこから読み始めるか

1. `<body>` タグ内の HTML 構造（タイトル・ボタン）を確認
2. `new Phaser.Game({ ... })` の設定オブジェクトで初期設定を把握
3. `create()` 関数で初期描画の処理を確認
4. `update()` 関数で毎フレームの処理を確認
5. ボタンの `onclick` から各機能の入口を追う

### よく出てくるパターン

```js
// グラフィックのクリアと再描画（update()内で毎フレーム）
graphics.clear();
// ... drawXxx() を呼んで再描画
```

```js
// イージング（なめらかなアニメーション）
currentValue += (targetValue - currentValue) * 0.07;
```

```js
// 音声の遅延初期化
function getAudioCtx() {
  if (!audioCtx) audioCtx = new AudioContext();
  if (audioCtx.state === 'suspended') audioCtx.resume();
  return audioCtx;
}
```

### 改造・拡張するなら

- 背景（空・草の色）→ `create()` 内の `drawBackground()` 相当部分
- 遮断機の速度 → `armAngle += (armAngleTarget - armAngle) * 0.07` の係数
- 電車の速度 → `trainSpeed` の最大値 `4.5`
- ベル音のテンポ → `setInterval` の `700`（ミリ秒）
- 電車の車両数・色 → `drawTrain()` 内のループ処理
